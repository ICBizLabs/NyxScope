mbelib-neo is bundled in two forms in this distribution:

  1. As `mbe-neo.dll` inside ./dsd-neo-libs/ (alongside dsd-neo.exe).
     dsd-neo.exe loads it dynamically at runtime.

  2. **Statically linked into the `sdr-imbe-helper` executable** which
     ships as a sidecar binary alongside the Nyx Scope main app. The
     `sdr-imbe-helper` binary is the Nyx Scope project's GPL containment
     line: the Nyx Scope main binary spawns it as a child process and
     communicates with it exclusively over a stdin/stdout pipe boundary.

Because the GPL static link is contained inside `sdr-imbe-helper` and
the main Nyx Scope binary uses a subprocess boundary (recognized by GPL
§0/§2 as a valid arrangement), the Nyx Scope main binary itself contains
no GPL object code and is governed by the proprietary EULA at
../eula.md (see §9.4).
